# Workflow разработки

Цель: выполнять итерации строго по `doc/tasklist.md` и следовать `vision.md`.

## Правила работы
- Идти по итерациям строго по порядку, не перескакивать.
- Перед каждой итерацией: предложить решение и показать предполагаемые изменения/отрезки кода.
- Реализация только после явного согласования.
- После реализации: ждать подтверждения.
- Обновлять прогресс в `doc/tasklist.md` и отмечать выполненные задачи.
- Перед переходом к следующей итерации: согласовать переход.
- После каждой итерации: делать коммит в репозиторий.
