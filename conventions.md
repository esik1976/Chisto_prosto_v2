# Conventions

Этот документ задает правила разработки для code‑ассистента. Основание: `vision.md`. Детали и технические решения см. в `vision.md`.

## Принципы
- Следуй KISS: минимально необходимое, без оверинжиниринга.
- Сохраняй монолитную архитектуру и один код‑базис.
- Делай упор на рабочие сценарии MVP, улучшения — позже.
- Минимизируй зависимости и сложность.

## Технологические рамки
- Стек и структура проекта берутся из `vision.md`.
- Не вводи новые технологии/подходы без явного запроса.

## Качество кода
- Пиши простой, читаемый код без лишней абстракции.
- Делай минимальные тесты только для ключевых сценариев.

## Конфигурация и логирование
- Следуй правилам конфигурирования и логирования из `vision.md`.
